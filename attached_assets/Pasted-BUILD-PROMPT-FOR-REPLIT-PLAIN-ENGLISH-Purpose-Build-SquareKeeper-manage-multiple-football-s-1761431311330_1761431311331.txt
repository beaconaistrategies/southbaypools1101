BUILD PROMPT FOR REPLIT (PLAIN ENGLISH)

Purpose:
- Build "SquareKeeper": manage multiple football squares pools with an admin backend and a simple public board for entrants.

What the user can do (MVP):
1) Admin: Create and manage multiple contests at once (each contest has a 10x10 board numbered 1–100, quarter info, team labels, and editable cells). Set how many header rows/columns are marked red (1–4 each).
2) Entrant: Click any available square, enter name, email, and entry name, and confirm. The chosen square is instantly grayed out/marked unavailable on the master board.
3) Admin: Edit or override square assignments, lock/unlock a contest, and optionally randomize or edit the 0–9 axis numbers. Admin can also mark red rows/columns per contest.

Pages & Navigation:
- Admin Dashboard: appears after admin signs in. Shows a list of contests with status (Open/Locked), quick stats (# taken / # available), and buttons: "New Contest", "Manage".
- New/Edit Contest: fields for contest details (name, event date, notes), team labels (Top Team, Left Team), quarter labels (Q1–Q4), axis numbers (0–9 for top and left; can randomize or type), red header settings (Red Rows: 1–4, Red Columns: 1–4; pick which rows/cols are red), and a toggle for "Open for Picks". Buttons: Save, Cancel.
- Contest Manager (Admin View): shows the 10x10 board (cells 1–100) with colors: gray=taken, white=available, red headers as chosen. Sidebar shows "Squares" list with filters (All/Available/Taken). Actions on a selected square: Assign/Edit (name, email, entry name), Release square, Mark Paid (optional), Notes. Top actions: Lock Contest, Unlock Contest, Shuffle Axis Numbers, Save Changes.
- Public Contest Board (Entrant View): shows contest name, top/left team labels, the 10x10 board, legend, and a simple form modal that appears when an available square is clicked (Name*, Email*, Entry Name*). Buttons: Confirm, Cancel. After confirming, the square grays out for everyone.
- Winners & Quarters (optional display on both views): small panel with Q1–Q4 text fields for winner square numbers (editable by admin only).

Data to remember:
- Contest {
    id, name, eventDate, status (open|locked),
    topTeam, leftTeam,
    topAxisNumbers[10], leftAxisNumbers[10],  // digits 0–9 in some order
    redRowsCount (1–4), redColsCount (1–4),
    redRows[] (indices 0–9), redCols[] (indices 0–9),
    notes
  }
- Square {
    contestId, index (1–100), row (0–9), col (0–9),
    status (available|taken),
    holderName, holderEmail, entryName,
    reservedAt
  }
- Quarters {
    contestId,
    q1WinnerSquare (1–100 | empty),
    q2WinnerSquare,
    q3WinnerSquare,
    q4WinnerSquare
  }
- Required fields:
  - For Admin creating a contest: name, eventDate, topTeam, leftTeam, redRowsCount, redColsCount.
  - For Entrant taking a square: holderName, holderEmail, entryName.
- Start with example records so the app looks real on first run.

Rules & Limits:
- Exactly 10x10 = 100 squares per contest (indexed 1–100). Cell numbers are visible on the board.
- Red header settings: Admin can choose between 1 and 4 red rows and 1 and 4 red columns for each contest, and pick which specific rows/columns are red. Red highlighting is visible on the board headers only.
- When an entrant confirms a square, change it to "taken" immediately and gray it out for everyone. Show the entrant their chosen square number.
- Prevent empty required fields; show friendly inline errors.
- Email must look like a valid email address.
- One click = one reservation. (Admin can later edit or release it.)
- Admin can lock a contest to stop new picks.
- Confirmation before deleting a contest or releasing a taken square.
- Max 200 contests total; each contest capped at 100 squares.

Safety & Access:
- Admin-only backend behind simple sign-in (email + password).
- Public entrants do not need accounts; they only enter name, email, and entry name to claim a square.
- Keep admin pages hidden from non-signed-in users. Public only sees the Contest Board page(s) that are Open.

Look & Feel:
- Clean, modern, readable layout.
- Top bar with app name on the left and the primary action on the right (e.g., "New Contest" on Admin Dashboard).
- Clear labels and helper text on forms (e.g., “Entry name appears on the board.”).
- Board cells show: small cell number, and if taken, show entry name (truncated) with a tooltip on hover for full details.
